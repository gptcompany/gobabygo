<task type="auto">
  <name>Task 1: Retry policy module</name>
  <files>src/router/retry.py</files>
  <action>
  1. Create `src/router/retry.py` with `RetryPolicy` class:

  2. **Configuration**:
     - `RetryPolicy.__init__(self, db: RouterDB, max_attempts: int = 3, backoff_schedule: list[int] | None = None, escalation_callbacks: list[EscalationCallback] | None = None, unschedulable_timeout_s: int = 1800)`
     - `backoff_schedule` default: `[15, 60, 180]` (seconds, indexed by attempt-1, i.e., attempt 1 retry waits 15s, attempt 2 waits 60s, attempt 3 waits 180s)
     - `unschedulable_timeout_s` default: 1800 (30 minutes)

  3. **EscalationCallback protocol** (typing.Protocol):
     - `class EscalationCallback(Protocol):`
       - `def on_escalation(self, task: Task, last_worker_id: str | None, attempt: int, reason: str) -> None: ...`
     - `class LogEscalation:` — default implementation that just logs via `logging.warning()`
     - Users can implement custom callbacks (e.g., Discord webhook, email alert)

  4. **Retry with backoff**:
     - `should_retry(self, task: Task) -> bool`: returns True if task.attempt < max_attempts
     - `calculate_not_before(self, task: Task) -> str`: returns ISO-8601 timestamp = now() + backoff_schedule[task.attempt - 1]. If attempt exceeds schedule length, use last value.
     - `requeue_with_backoff(self, task_id: str, reason: str, conn: sqlite3.Connection | None = None) -> RetryResult`:
       a. Get task
       b. If should_retry(task):
          - Calculate not_before
          - Within transaction (or using passed conn):
            - CAS update task status to 'queued' (from current status, using recovery-style direct CAS since these are abnormal transitions like timeout->queued)
            - Update task fields: attempt+1, assigned_worker=None, lease_expires_at=None, not_before=calculated
          - Emit event: 'task_retried' with attempt, not_before, reason
          - Return RetryResult(retried=True, new_attempt=task.attempt+1, not_before=not_before)
       c. If not should_retry(task):
          - CAS update task status to 'failed' (terminal)
          - Emit event: 'task_max_retries_exhausted' with attempt, reason
          - Invoke all escalation callbacks: callback.on_escalation(task, task.assigned_worker, task.attempt, reason)
          - Return RetryResult(retried=False, escalated=True, new_attempt=task.attempt)

  5. **RetryResult** dataclass:
     - `retried: bool`
     - `escalated: bool = False`
     - `new_attempt: int`
     - `not_before: str | None = None`
     - `error: str | None = None`

  6. **Task unschedulable detection**:
     - `find_unschedulable_tasks(self) -> list[Task]`:
       a. Query tasks WHERE status='queued' AND created_at < now() - unschedulable_timeout_s
       b. Return list of tasks that have been queued longer than the timeout
     - `emit_unschedulable_events(self) -> int`:
       a. For each unschedulable task, emit event 'task_unschedulable' with idempotency key based on task_id + date (so it's emitted at most once per day per task)
       b. Return count of events emitted
       c. v1: event only, no auto-action (monitoring can trigger alerts)
  </action>
