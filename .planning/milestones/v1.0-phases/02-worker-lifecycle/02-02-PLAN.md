<task type="auto">
  <name>Task 1: Scheduler module — deterministic selection + dispatch</name>
  <files>src/router/scheduler.py</files>
  <action>
  1. Create `src/router/scheduler.py` with `Scheduler` class:

  2. **Initialization**:
     - `Scheduler.__init__(self, db: RouterDB, lease_duration_s: int = 300)`: lease_duration_s is how long a lease is valid (default 5 minutes)

  3. **Worker selection** (deterministic):
     - `find_eligible_worker(self, task: Task) -> Worker | None`:
       a. Get all workers with `status='idle'` via `db.list_workers(status='idle')`
       b. Filter: `worker.cli_type == task.target_cli`
       c. Filter: `worker.account_profile == task.target_account` (exact match — KISS spec requires both target_cli AND target_account routing)
       d. Sort remaining by `idle_since ASC` (longest idle first = fairest round-robin)
       e. Return first match, or None if no eligible worker

  4. **Task selection** (next schedulable):
     - `find_next_task(self) -> Task | None`:
       a. Query tasks with `status='queued'` ordered by `priority DESC, created_at ASC`
       b. Filter: skip tasks where `not_before IS NOT NULL AND not_before > now()` (retry backoff)
       c. Filter: skip tasks where `depends_on` contains non-terminal task IDs (use dependency.check_dependencies)
       d. Return first schedulable task, or None

  5. **Dispatch** (atomic assignment with TOCTOU protection):
     - `dispatch(self) -> DispatchResult | None`:
       a. Call `find_next_task()` — if None, return None (nothing to dispatch)
       b. Call `find_all_eligible_workers(task) -> list[Worker]` — returns full sorted list (idle_since ASC). If empty, return None.
       c. Iterate through the pre-sorted eligible workers list:
          - For each candidate worker, within `db.transaction()`:
            - **CAS verify worker still idle**: `UPDATE workers SET status='busy' WHERE worker_id=? AND status='idle'`. If rowcount==0, this worker became stale/busy — skip to next candidate in list.
            - If CAS succeeded: Apply FSM transition `queued -> assigned`, create lease, update task fields.
            - Return `DispatchResult(task, worker, lease)` on success.
       d. If all candidates exhausted (all CAS failed), return None — task stays queued for next dispatch cycle.
       e. No re-query needed: the pre-sorted list from step (b) is iterated sequentially.

  6. **Task ACK** (worker confirms it started working):
     - `ack_task(self, task_id: str, worker_id: str) -> bool`:
       a. Get task — verify task.assigned_worker == worker_id
       b. Apply FSM transition: `assigned -> running` via fsm.apply_transition()
       c. Return success bool

  7. **Task completion** (worker reports done):
     - `complete_task(self, task_id: str, worker_id: str) -> bool`:
       a. Get task — verify task.assigned_worker == worker_id
       b. Within db.transaction():
          - Apply FSM transition: `running -> review` (or `running -> completed` if no verifier gate needed — Phase 2 uses `running -> completed` directly since verifier gate is Phase 3)
          - Expire lease
          - Update worker status: `busy -> idle`, set `idle_since=now()`
       c. Trigger `dependency.on_task_terminal()` to unblock dependent tasks
       d. Return success bool

  8. **Task failure** (worker reports failure):
     - `report_failure(self, task_id: str, worker_id: str, reason: str) -> bool`:
       a. Get task — verify task.assigned_worker == worker_id
       b. Within db.transaction():
          - Apply FSM transition: `running -> failed`
          - Expire lease
          - Update worker: `busy -> idle`, set `idle_since=now()`
       c. Trigger `dependency.on_task_terminal()`
       d. Return success bool

  9. **DispatchResult** dataclass:
     - `task: Task`
     - `worker: Worker`
     - `lease: Lease`
