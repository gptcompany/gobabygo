# Requirements: v1.4 Native Cross-Repo Handoff

**Defined:** 2026-03-04
**Status:** SHIPPED (2026-03-05)

## Topology

- [x] **TOPO-01**: Manual topology config defines `global`, `hosts`, `workers`, and `repos`
- [x] **TOPO-02**: Every repo maps to exactly one `REPO_LEAD` and a bounded worker pool
- [x] **TOPO-03**: Cross-repo routing policy is explicit and defaults to `require_president_handoff=true`
- [x] **TOPO-04**: Existing env-based per-host runtime config remains the deployment mechanism; topology adds orchestration metadata only
- [x] **TOPO-05**: Scheduler becomes topology-aware by constraining eligible workers through `repo -> worker_pool` when `task.repo` is set, without adding `repo` to the `Worker` model

## Native Attach

- [x] **ATCH-01**: Every `execution_mode=session` task publishes an attach handle
- [x] **ATCH-02**: `Session.metadata` carries `attach_kind` and `attach_target` with no new session table
- [x] **ATCH-03**: `meshctl` can read and display attach handles using the existing router API
- [x] **ATCH-04**: The current `/sessions/*` bus is explicitly not used as the primary interactive transport

## Notifications

- [x] **NOTIFY-01**: Matrix bridge emits notifications for approval/input/block/fail/complete events
- [x] **NOTIFY-02**: Notifications include repo identity and attach handle
- [x] **NOTIFY-03**: Quick text replies are fallback only; the primary human control path is terminal attach
- [x] **NOTIFY-04**: Notification bridge is out-of-process and can fail without stopping task execution

## Structured Handoff

- [x] **HANDOFF-01**: Cross-repo handoff packet includes `source_repo`, `target_repo`, `summary`, and `question`
- [x] **HANDOFF-02**: Cross-repo handoff packet includes `decisions`, `artifacts`, `open_risks`, and `related_session_ids`
- [x] **HANDOFF-03**: `PRESIDENT_GLOBAL` is the only cross-repo handoff producer/consumer in the initial rollout
- [x] **HANDOFF-04**: First iteration reuses existing task payload/event structures
- [x] **HANDOFF-05**: `thread_context` remains available and is not replaced by the handoff packet

## Shared Memory

- [x] **MEM-01**: OpenMemory runs as a separate self-hosted memory service
- [x] **MEM-02**: OpenMemory is consumed through its native MCP server as a shared memory layer
- [x] **MEM-03**: OpenMemory stores context and recall only; it is not a message bus or source of operational truth
- [x] **MEM-04**: Router operations remain correct when OpenMemory is unavailable
