### THRD-04 Detail

**Requirement:** "Session spawner: router creates tmux sessions on-demand when thread step activates (POST /sessions/spawn)"

**Implementation:** `session_spawner.py` provides `spawn_tmux_session`, `kill_tmux_session`, `is_session_alive` — fully tested (4 tests). However, no production `src/` module imports it. The router does NOT create tmux sessions; sessions are worker-owned via `session_worker.py`.

**Disposition:** Intentional architectural decision. Phase 15 VERIFICATION.md documents this as decision C: "thread_context as top-level field separate from payload (runtime enrichment, not persisted)" and SC3 verifies "Per step session, il runtime interattivo resta worker-owned e il router non duplica sessioni tmux." The requirement wording is outdated; the behavior is correct.
