## Integration Issues (1 High)

### INT-01: Missing /tasks/ack HTTP Endpoint

**Severity:** High (blocks production E2E)
**Files:** `src/router/server.py`, `src/router/worker_client.py`

`scheduler.ack_task()` exists and is tested (line 166), FSM supports `assigned -> running`, but no HTTP handler exposes it. `worker_client._execute_task()` goes straight to `/tasks/complete` without acking first, causing a 409 (CAS expects `running` but task is still `assigned`).

**Impact:** Tasks cycle: assigned -> lease expires -> recovery requeues -> assigned again (never completes).

**Fix:** Add `/tasks/ack` POST handler in server.py; add ack call in worker_client before execute.

## Warnings (3 Non-Blocking)

1. **heartbeat.py:187** — `Worker` type annotation not imported at module level (safe at runtime, fails mypy/static analysis)

3. **YAML mapping** — no rules for `gsd:implement-*` commands; resolves to `step=None`
