**models.py additions:**

1. Add `ThreadStatus` enum:
```python
class ThreadStatus(str, Enum):
    pending = "pending"    # created, no step dispatched yet
    active = "active"      # at least one step running/queued
    completed = "completed"  # all steps in terminal completed state
    failed = "failed"      # at least one step failed/timeout/canceled
```

### Task 2: Thread Module (`src/router/thread.py`)

**File**: `src/router/thread.py` (NEW, ~120 LOC)

1. `create_thread(db, name) -> Thread`:
- Create Thread model with UUID, insert via db.insert_thread()
- Return created thread

2. `add_step(db, thread_id, step_request: ThreadStepRequest) -> Task`:
- Validate thread exists and is not in terminal state (completed/failed)
- Create Task with thread fields set:
  - `thread_id = thread_id`
  - `step_index = step_request.step_index`
  - `repo = step_request.repo`
  - `role = None` (default, can be overridden)
- If step_index > 0, auto-set `depends_on` to previous step's task_id
  (query: `SELECT task_id FROM tasks WHERE thread_id = ? AND step_index = ?`, step_index - 1)
- If depends_on provided explicitly in request, use those instead (override)
- Set initial status to `blocked` if depends_on is non-empty, `queued` otherwise
- Insert task via db.insert_task()
- Return created task

3. `get_thread_context(db, thread_id, up_to_step_index) -> list[dict]`:
- Query completed steps: `thread_id = ? AND step_index < ? AND status = 'completed'` ORDER BY step_index
- For each, deserialize result_json
- Aggregate into list: `[{"step_index": N, "repo": "...", "result": {...}}, ...]`
- Apply 32KB cap: if aggregate JSON > 32KB, drop oldest results first
- Return list

4. `compute_thread_status(db, thread_id) -> ThreadStatus`:
- Query all steps for thread
- If no steps: `pending`
- If any step in failed/timeout/canceled: `failed`
- If all steps completed: `completed`
- If any step exists (running/assigned/queued/blocked): `active`
  (blocked steps mean the thread is in-flight waiting on dependencies, not "pending")
- Only `pending` if thread has zero steps

### Task 3: Session Spawner (`src/router/session_spawner.py`)

**File**: `src/router/session_spawner.py` (NEW, ~80 LOC)

1. `_sanitize_session_name(name: str) -> str`:
- Validate against `^[a-zA-Z0-9_-]+$`
- If invalid chars, strip them
- Return sanitized name

2. `spawn_tmux_session(thread_id: str, step_index: int, cli_command: str, work_dir: str = "") -> str`:
- Session name: `mesh-{thread_id[:8]}-s{step_index}`
- Sanitize session name
- Build command as list (NO shell=True): `['tmux', 'new-session', '-d', '-s', session_name]`
- If cli_command: tokenize via `shlex.split(cli_command)` (same pattern as Phase 13 CLI invocation), append as initial command
- If work_dir: add `-c work_dir` flag (work_dir validated as path, not shell-escaped)
- `subprocess.run(cmd, check=True, capture_output=True, timeout=10)`
- Return session_name on success
- Raise RuntimeError on failure

**Note on placement**: session_spawner runs in the router process, which shares the same machine
as the workers (single-VPS topology, see PROJECT.md Constraints). This is a conscious design decision
for the current architecture. If multi-host deployment is needed in the future, tmux spawning
would move to the worker side.

3. `kill_tmux_session(session_name: str) -> bool`:
- `subprocess.run(['tmux', 'kill-session', '-t', session_name], capture_output=True, timeout=5)`
- Return True if killed, False if not found (non-zero exit but no exception)

4. `is_session_alive(session_name: str) -> bool`:
- `subprocess.run(['tmux', 'has-session', '-t', session_name], capture_output=True, timeout=5)`
- Return exit code == 0

### Task 4: Scheduler Integration

**File**: `src/router/scheduler.py`

1. In `_route_to_completed()`, INSIDE the transaction (after result persistence, before lease expire):

```python
# Update thread status if this task belongs to a thread
if task.thread_id:
    from src.router.thread import compute_thread_status
    from src.router.db import _utc_now
    new_thread_status = compute_thread_status(self._db, task.thread_id)
    self._db.update_thread(
        task.thread_id,
        {"status": new_thread_status.value, "updated_at": _utc_now()},
        conn=conn,
    )
```

2. In `_route_to_review()`, same pattern as above (inside transaction).

3. In `report_failure()`, same pattern (inside transaction).

4. In `_try_dispatch()`, INSIDE the transaction: if task has thread_id and thread.status == "pending", transition to "active":
```python
if task.thread_id:
    thread = self._db.get_thread(task.thread_id)
    if thread and thread.status.value == "pending":
        self._db.update_thread(
            task.thread_id,
            {"status": "active", "updated_at": _utc_now()},
            conn=conn,
        )
```
