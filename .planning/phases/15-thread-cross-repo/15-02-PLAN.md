## Tasks

### Task 1: Server Thread Endpoints

**File**: `src/router/server.py`

**New GET routes** (add to `do_GET`):

1. `GET /threads` → `_handle_list_threads()`
- Query params: `status` (optional), `name` (optional, exact match), `limit` (default 50)
- Returns `{"threads": [thread.model_dump()...]}`
- `name` filter is needed for meshctl `_resolve_thread_id` name-to-id lookup

2. `GET /threads/{id}` → `_handle_get_thread(thread_id)`
- 404 if not found
- Returns thread.model_dump()

3. `GET /threads/{id}/status` → `_handle_thread_status(thread_id)`
- Returns thread + all steps with status:
```json
{
  "thread": {...},
  "steps": [
    {"step_index": 0, "task_id": "...", "status": "completed", "repo": "...", "title": "..."},
    {"step_index": 1, "task_id": "...", "status": "running", "repo": "...", "title": "..."}
  ]
}
```

4. `GET /threads/{id}/context` → `_handle_thread_context(thread_id)`
- Calls `thread.get_thread_context(db, thread_id, step_index=999)`
- Returns `{"thread_id": "...", "context": [...]}`

**New POST routes** (add to `do_POST`):

5. `POST /threads` → `_handle_create_thread()`
- Body: `{"name": "..."}`
- Validates via ThreadCreateRequest
- Calls `thread.create_thread(db, name)`
- Returns 201 `{"status": "created", "thread_id": "...", "name": "..."}`

6. `POST /threads/{id}/steps` → `_handle_add_step(thread_id)`
- Body: ThreadStepRequest fields
- Validates via ThreadStepRequest
- Calls `thread.add_step(db, thread_id, step_request)`
- Returns 201 `{"status": "created", "task_id": "...", "step_index": N}`

**Route matching update:**

7. Update `do_GET` routing to handle thread paths:
```python
elif path == "/threads":
    self._handle_list_threads()
elif path.startswith("/threads/"):
    # Parse: /threads/{id}, /threads/{id}/status, /threads/{id}/context
    parts = path[len("/threads/"):].split("/")
    thread_id = parts[0]
    if len(parts) == 1:
        self._handle_get_thread(thread_id)
    elif len(parts) == 2 and parts[1] == "status":
        self._handle_thread_status(thread_id)
    elif len(parts) == 2 and parts[1] == "context":
        self._handle_thread_context(thread_id)
    else:
        self._send_json(404, {"error": "not_found"})
```

8. Update `do_POST` routing:
```python

    self._handle_create_thread()
elif path.startswith("/threads/") and path.endswith("/steps"):
    thread_id = path[len("/threads/"):-len("/steps")]
    if thread_id:
        self._handle_add_step(thread_id)
    else:
        self._send_json(404, {"error": "not_found"})
```

### Task 2: Thread Context Enrichment in Long-Poll

**File**: `src/router/server.py`

1. In `_handle_task_poll()`, after getting the result task, enrich with thread_context:

```python
if result.task is not None:
    task_dict = result.task.model_dump(mode="json")
    # Runtime thread context enrichment
    if result.task.thread_id and result.task.step_index and result.task.step_index > 0:
        from src.router.thread import get_thread_context
        thread_ctx = get_thread_context(db, result.task.thread_id, result.task.step_index)
        task_dict["thread_context"] = thread_ctx
    self._send_json(200, task_dict)
```

Key points:
- `thread_context` is a **top-level field** separate from `payload`
- Only enriched when task has thread_id AND step_index > 0 (step 0 has no previous context)
- Computed on-the-fly, NOT persisted to DB
- 32KB cap enforced by `get_thread_context()`

### Task 3: Session Spawner Integration

**File**: `src/router/server.py`

1. In `_handle_task_ack()`, after successful ack (task assigned → running), spawn tmux session
if task belongs to a thread. **Use fire-and-forget thread to avoid blocking the HTTP response**:

```python
if ok:
    task = db.get_task(task_id)
    if task and task.thread_id and task.step_index is not None:
        import threading as _threading
        def _spawn():
            from src.router.session_spawner import spawn_tmux_session
            try:
                cli_cmd = os.environ.get("MESH_CLI_COMMAND", "claude --print -p")
                work_dir = task.repo or ""
                session_name = spawn_tmux_session(
                    task.thread_id, task.step_index, cli_cmd, work_dir
                )
                logger.info("tmux session spawned: %s for task %s", session_name, task_id)
            except Exception as e:
                logger.warning("tmux spawn failed for task %s: %s", task_id, e)
        _threading.Thread(target=_spawn, daemon=True).start()
    self._send_json(200, {"status": "acknowledged"})
```

**Rationale**: tmux spawn (subprocess with up to 10s timeout) must not block the HTTP handler.
Fire-and-forget daemon thread ensures the response is immediate. Failure is logged but does not
affect task execution.

2. In `_handle_task_complete()` and `_handle_task_fail()`, cleanup tmux session
if task belongs to a thread:

```python
# After successful completion/failure handling
task = db.get_task(task_id)
if task and task.thread_id and task.step_index is not None:
    from src.router.session_spawner import kill_tmux_session
    session_name = f"mesh-{task.thread_id[:8]}-s{task.step_index}"
    kill_tmux_session(session_name)
```

def cmd_thread_create(args):
    base = _base_url()
    headers = _headers()
    headers["Content-Type"] = "application/json"
    body = {"name": args.name}
    resp = requests.post(f"{base}/threads", json=body, headers=headers, timeout=10)
    if resp.status_code == 201:
        data = resp.json()
        print(f"Thread created: {data.get('thread_id', '?')} ({data.get('name', '')})")
    else:
        print(f"Error: {resp.status_code} -- {resp.text}", file=sys.stderr)
        sys.exit(1)

def cmd_thread_add_step(args):
    base = _base_url()
    headers = _headers()
    headers["Content-Type"] = "application/json"
    # Resolve thread_id from name if needed
    thread_id = _resolve_thread_id(args.thread)
    body = {
        "title": args.title,
        "step_index": args.step_index,
        "repo": args.repo or "",
        "target_cli": args.cli or "claude",

        sys.exit(1)

3. **`cmd_thread_status(args)`**:
```python
def cmd_thread_status(args):
    base = _base_url()
    headers = _headers()
    thread_id = _resolve_thread_id(args.thread)
    resp = requests.get(f"{base}/threads/{thread_id}/status", headers=headers, timeout=10)
    if resp.status_code != 200:
        print(f"Error: {resp.status_code}", file=sys.stderr)
        sys.exit(1)
    data = resp.json()
    thread = data["thread"]
    steps = data["steps"]
    if args.json_output:
        print(json.dumps(data, indent=2))
        return
    print(f"THREAD: {thread.get('name', '?')} [{thread.get('status', '?')}]")
    print(f"{'STEP':<6} {'STATUS':<12} {'REPO':<20} TITLE")
    for s in steps:
        idx = s.get("step_index", "?")
        status = s.get("status", "?")[:12]
        repo = (s.get("repo", "") or "")[:20]
        title = s.get("title", "")[:40]
        print(f"{idx:<6} {status:<12} {repo:<20} {title}")
```

4. **`cmd_thread_context(args)`**:

    resp = requests.get(f"{base}/threads/{thread_id}/context", headers=headers, timeout=10)
    if resp.status_code != 200:

        sys.exit(1)
    data = resp.json()
    print(json.dumps(data, indent=2))

5. **Helper `_resolve_thread_id(name_or_id)`**:
- If looks like UUID (contains dashes, 36 chars), use as-is (GET /threads/{id})
- Otherwise, treat as name: GET /threads?name={name_or_id}
  - If exactly 1 match: use that thread_id
  - If 0 matches: error "Thread not found"
  - If multiple matches: error "Ambiguous thread name '{name}', {N} threads match. Use thread_id instead."
- This handles the name collision case explicitly — user must use thread_id when names collide
