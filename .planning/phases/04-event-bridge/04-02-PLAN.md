<objective>
Implement the YAML-based mapping engine that maps GSD command names to semantic steps.

Purpose: The mapping engine provides a 3-layer semantic inference system: Layer A (auto base events — handled by emitter), Layer B (rule-based pattern matching from YAML), and Layer C (explicit overrides for critical commands). When a GSD command fires, the mapping engine determines the semantic `step` (research, discuss, plan, implement, review, remediation) from the command name, plus any override-specific behavior.

Output:
- `bridge/mapping.py`: MappingEngine class with `resolve(command_name) -> MappingResult`
- `mapping/command_rules.yaml`: Pattern rules with regex matching
- `mapping/command_overrides.yaml`: Critical command overrides
- Comprehensive tests
</objective>

    matched_rule: str | None  # the regex pattern that matched

class MappingEngine:
    def __init__(self, rules_path: str | Path, overrides_path: str | Path | None = None):
        """Load YAML rules and overrides on init."""

    def resolve(self, command_name: str) -> MappingResult:
        """Resolve command name to semantic step.

        Priority:
        1. Check overrides first (exact command match)
        2. Check rules (regex pattern match, first match wins)
        3. Return MappingResult(step=None) if no match (Layer A auto-event still works)
        """
