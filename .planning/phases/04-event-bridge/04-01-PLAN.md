<objective>
Implement the core event emitter with CloudEvent envelope creation, JSON Schema validation, and pluggable transport layer.

Purpose: Build the foundational event bridge components — the EventEmitter wraps GSD command activity into CloudEvent-compliant envelopes, validates them against a JSON Schema, and dispatches via a transport adapter. Two transport implementations: InProcessTransport (direct DB write for testing/dev) and HttpTransport (HTTP POST for VPN production).

Output:
- `bridge/emitter.py`: EventEmitter class with `emit(command, event_kind, ...)` method
- `bridge/transport.py`: EventTransport protocol + InProcessTransport + HttpTransport
- `bridge/schema.py`: JSON Schema loading + validation via Draft202012Validator
- `schemas/command_event.schema.json`: CloudEvent data schema (Draft 2020-12)
- CommunicationPolicy integration: validate sender role before accepting events
- Comprehensive tests for emitter, transport, and schema validation
</objective>

## 2. JSON Schema: `schemas/command_event.schema.json`

Draft 2020-12 schema for CloudEvent data payload:
```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["run_id", "gsd_command", "event", "idempotency_key", "ts"],
  "properties": {
    "run_id": {"type": "string", "format": "uuid"},
    "task_id": {"type": ["string", "null"]},
    "phase": {"type": ["string", "null"]},
    "gsd_command": {"type": "string", "minLength": 1},
    "step": {"type": ["string", "null"], "enum": [null, "research", "discuss", "plan", "implement", "review", "remediation"]},
    "event": {"type": "string", "enum": ["started", "completed", "failed"]},
    "target_cli": {"type": ["string", "null"], "enum": [null, "claude", "codex", "gemini"]},
    "target_account": {"type": ["string", "null"]},
    "status": {"type": ["string", "null"]},
    "attempt": {"type": "integer", "minimum": 1},
    "idempotency_key": {"type": "string", "minLength": 1},
    "artifact_paths": {"type": "array", "items": {"type": "string"}},
    "ts": {"type": "string", "format": "date-time"},
    "duration_ms": {"type": ["integer", "null"], "minimum": 0},
    "sender_role": {"type": "string", "enum": ["boss", "president", "worker"]}
  },
  "additionalProperties": false
}
```

        sender_role: str = "president",

        phase: str | None = None,
        step: str | None = None,
        target_cli: str | None = None,
        target_account: str | None = None,
        status: str | None = None,

        artifact_paths: list[str] | None = None,
        duration_ms: int | None = None,
    ) -> bool:
        """Create CloudEvent, validate, check CommPolicy, send via transport.

        1. If comm_policy set, validate sender_role can create events
        2. Build data dict from parameters

        4. Validate data against JSON Schema
        5. Create CloudEvent with type="com.mesh.command.{event_kind}", source="mesh/gsd-bridge/{machine}"
        6. Serialize to JSON
        7. Send via transport
        8. Return success/failure
