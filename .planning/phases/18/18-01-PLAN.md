# Phase 18A: Session Attach Lifecycle

## Scope

Add per-session attach handle to `session_worker.py`. Each interactive session
(`execution_mode=session`) gets an attach target in `Session.metadata` so
operators/bridges can connect.

**Primary path:** `upterm host` as worker-owned child process per session.
**Fallback:** `ssh_tmux` (static user@host pointer to the existing tmux session).
**Degraded:** no attach available -- task continues unaffected.

## Files

| File | Action |
|------|--------|
| `src/router/session_worker.py` | Modify -- config + attach lifecycle |
| `tests/router/test_session_worker.py` | Modify -- new tests |

**Not touched:** `server.py`, `db.py`, `fsm.py`, `scheduler.py`, `meshctl.py`, models.

## Design

### Config additions (SessionWorkerConfig)

- `upterm_bin` -- path to upterm (default `"upterm"`)
- `upterm_server` -- optional `--server` flag for upterm
- `upterm_ready_timeout` -- seconds to wait for upterm session URL (default `10.0`)
- `ssh_tmux_user` / `ssh_tmux_host` -- for ssh_tmux fallback

### Attach flow (in _execute_task)

1. Create tmux session (existing)
2. **NEW** `_create_attach_handle(tmux_session)` -> `(metadata | None, Popen | None)`
   - tries `_start_upterm()` first
   - falls back to ssh_tmux if configured
   - returns None if neither available
3. `_open_session()` receives attach metadata -- merged into Session.metadata
4. Session poll loop runs as before
5. **NEW** `finally:` block calls `_stop_upterm()` on the Popen

Note: upterm starts BEFORE `_open_session()` because there is no metadata
update endpoint. This avoids touching `server.py`.

### Metadata contract

```python
Session.metadata = {
    "tmux_session": "mesh-claude-work-abc12345",  # preserved
    "working_dir": "/tmp/mesh-tasks",              # preserved
    "task_title": "...",                           # preserved
    "attach_kind": "upterm",                       # NEW
    "attach_target": "ssh://TOKEN@host:port",      # NEW
}
```

### Failure handling

| Scenario | Behavior |
|----------|----------|
| upterm binary not found | fallback or no-attach |
| upterm starts but no URL within timeout | kill, fallback |
| upterm dies mid-session | attach becomes stale, task continues |
| ssh_tmux not configured + upterm fails | task continues, no attach_kind |
| session close/error/timeout | upterm killed via SIGTERM then SIGKILL |

## Acceptance

- [x] Every `execution_mode=session` task gets `attach_kind` + `attach_target` in
  Session.metadata OR continues without
- [x] upterm child process is always cleaned up
- [x] No key material in Worker model or /register
- [x] Session bus semantics unchanged
- [x] Existing metadata fields preserved

## Test strategy

All upterm tests mock subprocess (binary not available in CI/dev).
Tests cover:

1. `_parse_upterm_ssh_url` -- pure parsing
2. `_create_attach_handle` -- upterm success path
3. `_create_attach_handle` -- upterm fail -> ssh_tmux fallback
4. `_create_attach_handle` -- no attach available
5. `_stop_upterm` -- terminate + kill
6. `_open_session` metadata merging
7. Cleanup in finally block
