  <files>src/router/recovery.py</files>
  <action>
  1. Create `src/router/recovery.py` with:
     - `RecoveryResult` dataclass: tasks_requeued (int), leases_expired (int), events_replayed (int), errors (list[str])
     - `recover_on_startup(db: RouterDB, max_attempts: int = 3) -> RecoveryResult`:
       a. Find all active leases where `expires_at < now()` → expire each lease
       b. For each expired lease: get the associated task
          - If task.attempt < max_attempts: update task status to `queued` (via FSM apply_transition from wherever it is), increment attempt
          - If task.attempt >= max_attempts: transition to `failed` (terminal), create escalation event
       c. Find tasks in `assigned` or `running` status with no active lease → same requeue logic
       d. Log all recovery actions as TaskEvent entries
       e. Return RecoveryResult with counts
     - `audit_timeline(db: RouterDB, task_id: str) -> list[TaskEvent]`: replay events for a task in chronological order (for debugging)
  2. Recovery MUST use FSM apply_transition (not direct db.update_task_status) to maintain transition guard integrity.
  3. Recovery is idempotent: running it twice on the same state produces the same result (no double-requeue).
  </action>
  <verify>python -c "from src.router.recovery import recover_on_startup, RecoveryResult; print('Recovery OK')"</verify>
  <done>Recovery requeues expired tasks, respects max_attempts, uses FSM for all transitions</done>

<task type="auto">
  <name>Task 2: Event-driven dependency resolution</name>
  <files>src/router/dependency.py</files>
  <action>
  1. Create `src/router/dependency.py` with:
     - `check_dependencies(db: RouterDB, task_id: str) -> tuple[bool, list[str]]`:
       Returns (all_resolved: bool, unresolved_task_ids: list)
       A dependency is resolved when the depended-on task is in a terminal state (completed, failed, canceled).
     - `resolve_blocked_tasks(db: RouterDB) -> int`:
       Find all tasks with status=blocked.
       For each, check_dependencies. If all resolved → transition blocked->queued (via FSM).
       Return count of unblocked tasks.
     - `on_task_terminal(db: RouterDB, completed_task_id: str) -> int`:
       Called when any task reaches terminal state.
       Find tasks that have completed_task_id in their depends_on list AND are in blocked status.
       For each, check if ALL dependencies now resolved. If yes → transition to queued.
       Return count of newly unblocked tasks.
       This is the event-driven trigger — no polling needed.
  2. `on_task_terminal` is the primary mechanism. `resolve_blocked_tasks` is a batch fallback for recovery scenarios.
  </action>
  <verify>python -c "from src.router.dependency import check_dependencies, on_task_terminal; print('Dependency OK')"</verify>
  <done>Dependencies resolved event-driven (on terminal), batch fallback for recovery, no polling</done>
</task>
