<task type="auto">
  <name>Task 1: FSM transition guard</name>
  <files>src/router/fsm.py</files>
  <action>
  1. Create `src/router/fsm.py` with:
     - `ALLOWED_TRANSITIONS: dict[TaskStatus, set[TaskStatus]]` mapping from CONTEXT.md FSM table:
       - queued -> {assigned, canceled}
       - assigned -> {blocked, running, canceled}
       - blocked -> {queued, canceled}
       - running -> {review, failed, timeout, canceled}
       - review -> {completed, failed, canceled}
       - Terminal states (completed, failed, timeout, canceled) -> {} (no transitions)
       NOTE: `canceled` is reachable from ALL non-terminal states per CONTEXT.md ("any non-terminal -> canceled").
     - `validate_transition(from_status: TaskStatus, to_status: TaskStatus) -> bool`: checks ALLOWED_TRANSITIONS
     - `TransitionRequest` dataclass: task_id, from_status, to_status, reason (str), timestamp
     - `apply_transition(db: RouterDB, request: TransitionRequest) -> TransitionResult`:
       a. Validate transition is allowed via ALLOWED_TRANSITIONS
       b. If invalid: write to dead-letter, return TransitionResult(success=False, reason="...")
       c. If valid: use `db.transaction()` context manager to atomically: (1) call db.update_task_status with CAS, (2) insert TaskEvent recording the transition. Both succeed or both rollback.
       d. If CAS fails (concurrent modification): write to dead-letter, return failure
       e. If CAS succeeds and event inserted: return success
     - `TransitionResult` dataclass: success (bool), reason (str | None), event_id (str | None)
  2. All transitions go through apply_transition — no direct db.update_task_status calls from outside.
  </action>
  <verify>python -c "from src.router.fsm import validate_transition, ALLOWED_TRANSITIONS; from src.router.models import TaskStatus; assert validate_transition(TaskStatus.queued, TaskStatus.assigned); assert not validate_transition(TaskStatus.queued, TaskStatus.completed); print('FSM OK')"</verify>
  <done>FSM validates all transitions from CONTEXT.md table, rejects invalid ones</done>
</task>
