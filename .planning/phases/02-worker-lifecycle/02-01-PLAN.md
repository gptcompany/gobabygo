     - `validate_token(self, token: str) -> bool`: check token exists in list AND (expires_at is None OR expires_at > now())
     - Grace period: during rotation, both old and new tokens are valid simultaneously

  3. **Registration**:
     - `register_worker(self, worker: Worker, token: str) -> tuple[bool, str]`: returns (success, message)
     - Validates token first — if invalid, return (False, "invalid_token")
     - Account uniqueness (atomic): wrap check+insert in `db.transaction()` to prevent TOCTOU race. Within the transaction: call `db.find_worker_by_account(worker.account_profile, exclude_statuses=['offline'])` — if found AND found.worker_id != worker.worker_id, rollback and return (False, "account_in_use")
     - Fast re-registration: if worker with same worker_id already exists AND is stale/offline, update status to idle + set idle_since=now + set last_heartbeat=now (via db.update_worker)
     - New registration: if worker_id not found, db.insert_worker with status='idle', idle_since=now
     - Return (True, "registered")

  4. **Deregistration**:
     - `deregister_worker(self, worker_id: str) -> tuple[bool, str]`: returns (success, message)
     - Get worker — if not found, return (False, "not_found")
     - If worker has active tasks (status='busy'), find active leases for this worker's tasks, expire them, requeue tasks using recovery-style CAS pattern (same as recovery.py: attempt+1, failed at max_attempts=3). This is a standalone implementation — no dependency on RetryPolicy module. Deregistration always counts as a retry attempt to prevent infinite requeue bypass.
     - Set worker status to 'offline' via db.update_worker
     - Emit 'worker_deregistered' event
     - Return (True, "deregistered")

  5. **Worker status FSM helper**:
     - `WORKER_TRANSITIONS`: dict mapping allowed worker status transitions:
       ```
       offline -> idle
       idle -> busy, stale, offline
       busy -> idle, stale, offline
       stale -> idle, offline
       ```
     - `transition_worker_status(self, worker_id, old_status, new_status) -> bool`: validates transition is allowed, then calls db.update_worker with new status + relevant timestamps (idle_since for ->idle, stale_since for ->stale)

<task type="auto">
  <name>Task 3: Heartbeat receiver + stale detection sweep</name>
  <files>src/router/heartbeat.py</files>
  <action>
  1. Create `src/router/heartbeat.py` with `HeartbeatManager` class:

  2. **Heartbeat receiver**:
     - `HeartbeatManager.__init__(self, db: RouterDB, stale_threshold_s: int = 35)`
     - `receive_heartbeat(self, worker_id: str) -> dict`: returns response dict
     - Get worker — if not found, return {"status": "unknown_worker"}
     - If worker.status == 'stale':
       - Transition stale -> idle (via db.update_worker: status='idle', idle_since=now, stale_since=None)
       - Find tasks that were requeued due to this worker's stale event (query events where event_type='worker_stale_requeue' AND payload contains worker_id, since last stale_since)
       - Return {"status": "stale_recovered", "requeued_tasks": [task_ids]} — worker MUST abort any running work
     - If worker.status in ('idle', 'busy'):
       - Update last_heartbeat=now via db.update_worker
       - Return {"status": "ok"}
     - If worker.status == 'offline': return {"status": "offline"} (worker should re-register)

  3. **Stale detection sweep**:
     - `run_stale_sweep(self, max_attempts: int = 3) -> SweepResult`: standalone implementation with no external dependencies
     - Calculate threshold: `now() - stale_threshold_s`
     - Query stale candidates via db.list_stale_candidates(threshold_iso)
     - For each stale candidate:
       a. Set worker status to 'stale', stale_since=now via db.update_worker
       b. Find all leases for this worker: query leases table WHERE worker_id = candidate.worker_id
       c. For each lease:
          - Get the task
          - If task.status in ('assigned', 'running'):
            - Expire lease via db.expire_lease
            - Use recovery-style CAS pattern (same as recovery.py): attempt+1 if attempt < max_attempts, otherwise failed (terminal). Clear assigned_worker and lease_expires_at.
            - Emit event: 'worker_stale_requeue' or 'worker_stale_max_attempts'
       d. Emit 'worker_stale' event with worker_id and task count
     - Use `db.transaction()` for atomicity per worker: the ENTIRE cleanup for one worker (mark stale + expire all its leases + requeue/fail all its tasks + emit events) runs in a SINGLE `db.transaction()` block. Partial failure rolls back all changes for that worker.
     - NOTE: The requeue logic here is a shared helper function `_requeue_task(db, task_id, max_attempts, reason, conn)` defined in a utility module (`src/router/_requeue.py`) or within heartbeat.py. Plan 02-03's RetryPolicy will IMPORT and REUSE this same helper (adding escalation callbacks on top) to avoid logic duplication.

  4. **SweepResult** dataclass:
     - `workers_marked_stale: int`
     - `tasks_requeued: int`
     - `tasks_failed: int`
     - `errors: list[str]`
  </action>
