# Phase 17: Role Topology + Manual Wiring

## Scope

Two sub-steps on the same branch (`feat-v1.4-phase17`):

| Sub | Deliverable | Files |
|-----|-------------|-------|
| 17A | Topology loader + validation | `src/router/topology.py` (new) |
| 17B | Scheduler topology-aware filter | `src/router/scheduler.py`, `src/router/server.py` |

## 17A ‚Äî Topology Loader

New module `src/router/topology.py`:

- Reads `MESH_TOPOLOGY_PATH` env var
- If absent ‚Üí topology disabled, all APIs return None/empty (legacy behavior)
- If present ‚Üí loads YAML once at import/init time, validates structure
- Required top-level keys: `version`, `global`, `hosts`, `workers`, `repos`
- On validation failure with env set ‚Üí raises `TopologyError` at startup

Public API:
- `load_topology(path: str | None) -> Topology | None`
- `Topology.get_repo_worker_pool(repo: str) -> list[str] | None`
- `Topology.get_repo_preferred_host(repo: str) -> str | None`
- `Topology.get_repo_notify_room(repo: str) -> str | None`
- `Topology.is_president_handoff_required() -> bool`

## 17B ‚Äî Scheduler Topology-Aware Filter

Modify `Scheduler.__init__` to accept optional `Topology`.
Modify `Scheduler.find_all_eligible_workers()`:

1. Keep existing filter order: `target_cli` ‚Üí `target_account` ‚Üí `execution_mode`
2. AFTER existing filters, if `task.repo` is set AND topology defines `repos.<repo>.worker_pool`:
   - Intersect eligible with that pool (by `worker_id`)
3. If `task.repo` is None or no pool defined ‚Üí full legacy fallback

Modify `server.py` bootstrap: load topology, pass to Scheduler.

## Acceptance Criteria

1. Without `MESH_TOPOLOGY_PATH` ‚Üí everything unchanged (zero behavioral diff)
2. With valid topology ‚Üí `task.repo` constrains to `repos.<repo>.worker_pool`
3. With invalid topology file + env set ‚Üí explicit `TopologyError` at startup
4. Topology missing `repos.<repo>` for a task's repo ‚Üí legacy fallback
5. No regressions on existing scheduler tests
