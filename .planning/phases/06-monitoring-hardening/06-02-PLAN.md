## Implementation

### Task 1: Grafana Alert Rules (Provisioning YAML)

**File**: `deploy/monitoring/mesh-alerts.yaml` (NEW)

Grafana provisioning format with 5 alert rules:

```yaml
apiVersion: 1
groups:
  - orgId: 1
    name: mesh-network
    folder: Mesh
    interval: 1m
    rules:
      - uid: mesh-router-down
        title: MeshRouterDown
        condition: C
        data:
          - refId: A
            datasourceUid: <VICTORIAMETRICS_UID>
            model:
              expr: mesh_router_up
              instant: true
          - refId: B
            datasourceUid: __expr__
            model:
              type: reduce
              reducer: last
              expression: A
          - refId: C
            datasourceUid: __expr__
            model:
              type: threshold
              expression: B
              conditions:
                - evaluator:
                    type: lt
                    params: [1]
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "Mesh router is down"
          description: "mesh_router_up < 1 for > 1m"

      - uid: mesh-worker-stale
        title: MeshWorkerStale
        # mesh_workers_stale > 0 for > 2m
        for: 2m
        labels:
          severity: warning

      - uid: mesh-queue-depth-high
        title: MeshQueueDepthHigh
        # mesh_queue_depth > 50 for > 5m
        for: 5m
        labels:
          severity: warning

      - uid: mesh-no-data
        title: MeshNoData
        # absent(mesh_router_up) for > 5m
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "No data from mesh router"
          description: "Scrape target not responding for > 5m"

      - uid: mesh-failure-rate-high
        title: MeshTaskFailureRateHigh
        # increase(mesh_tasks_failed_total[5m]) / (increase(mesh_tasks_completed_total[5m]) + increase(mesh_tasks_failed_total[5m])) > 0.05
        # Uses increase() instead of rate() for gauge-sourced totals
        # clamp_min denominator to avoid NaN on zero activity
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "Task failure rate > 5%"
          description: "SLO violation: success rate < 95% over 5m window"
```

**Note**: `<VICTORIAMETRICS_UID>` is a placeholder — user replaces with actual datasource UID from Grafana.
