# Plan 07-01: Register Validation via WorkerManager

**Phase:** 07-tech-debt-cleanup
**Requirement:** DEBT-01
**Goal:** server._handle_register() validates through WorkerManager (not just global bearer token)

## Current State

- `_handle_register()` in `server.py:156-172` calls `db.upsert_worker()` directly
- Bypasses `WorkerManager.register_worker()` which has: per-token auth, account uniqueness, re-registration logic, event emission
- `WorkerManager` is never instantiated in `run_server()` or `router_state`
- `_check_auth()` is a global bearer token check used by all endpoints

## Changes

### 1. Wire WorkerManager into server startup

**File:** `src/router/server.py` — `run_server()` function

- Instantiate `WorkerManager(db=db)` after DB creation
- Add to `router_state["worker_manager"]`
- Bridge `MESH_AUTH_TOKEN` env var into WorkerManager tokens:
  - If `MESH_AUTH_TOKEN` set: register it as a valid token in WorkerManager
  - If NOT set: require explicit `MESH_DEV_MODE=1` env var for open registration (fail-closed by default)
  - If neither set: reject all registrations with 403

### 2. Rewrite `_handle_register()` to use WorkerManager

**File:** `src/router/server.py` — `_handle_register()` method

Replace direct `db.upsert_worker()` call with `worker_manager.register_worker()`:

```python
def _handle_register(self) -> None:
    body = self._read_body()
    if body is None:
        return
    try:
        data = json.loads(body)
    except json.JSONDecodeError:
        self._send_json(400, {"error": "invalid_json"})
        return
    try:
        worker = Worker(**data)
    except (ValueError, TypeError, KeyError) as e:
        self._send_json(400, {"error": f"invalid_worker_data: {type(e).__name__}"})
        return

    # Extract bearer token from Authorization header
    auth_header = self.headers.get("Authorization", "")
    token = auth_header.removeprefix("Bearer ").strip() if auth_header else ""

    wm: WorkerManager = self.server.router_state["worker_manager"]
    success, message = wm.register_worker(worker, token)

    if not success:
        if message == "invalid_token":
            self._send_json(401, {"error": "invalid_token"})
        elif message == "account_in_use":
            self._send_json(409, {"error": "account_in_use"})
        else:
            self._send_json(400, {"error": message})
        return

    # Distinguish 200 (re-register) vs 201 (new)
    status_code = 200 if message == "re-registered" else 201
    self._send_json(status_code, {"status": "registered", "worker_id": worker.worker_id})
```

Note: Remove `_check_auth()` call from register path — WorkerManager handles auth.

### 3. Update WorkerManager.register_worker() return values

**File:** `src/router/worker_manager.py`

- Return `(True, "re-registered")` for re-registration of existing worker_id
- Return `(True, "registered")` for new worker
- Dev mode: if `MESH_DEV_MODE=1`, accept empty token (skip validate_token)

### 4. Keep `_check_auth()` for other endpoints

No change to `_check_auth()` — it continues to guard `/heartbeat`, `/tasks/*`, `/events` with the global bearer token. Auth unification deferred per CONTEXT.md.

### 5. Update tests

**File:** `tests/router/test_server.py` — `TestRegisterEndpoint`

Update existing tests and add new cases:
- Valid token → 201 Created
- Invalid/missing token → 401 Unauthorized
- Account already in use → 409 Conflict

- Invalid JSON → 400 (unchanged)
